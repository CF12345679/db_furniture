## dev_order.md（开发循序与测试点）

> 目标：农村红木家具广告网站，微信端（微信 H5）展示。先用占位数据跑通闭环，后续管理端上传完善内容。
>
> 当前已确定：
> - 落地形态：微信 H5
> - 商品规模：产品 `50-200`，分类 `8-20`（可先用占位数量替代）
> - 获客优先：微信咨询（二维码扫码 + 微信号复制添加），电话兜底
> - 线索：MVP 保留
> - 防刷：后端生成图片验证码（有效期 1 分钟）
>   - `captchaId` 过期返回文案：`验证码过期请重新获取并刷新图片`

可验证测试点

- 后端 `health` 接口可访问

### Step 1：内存数据 + 后端接口骨架（不接 MySQL）
先不建数据库，使用内存（List/Map）提供占位数据与接口骨架，确保前端/联调可跑通。

后端公开接口（先做返回占位数据/内存读写）：
- `GET /api/banners`
- `GET /api/categories`
- `GET /api/products/:productId`
- `GET /api/products?categoryId=...`
- `GET /api/company`
- `GET /api/cases`（如 MVP 可选）

可验证测试点
- 用 Postman/浏览器访问上述接口，返回 JSON 结构正确
- 数据能从内存读到（或至少能稳定返回占位）

### Step 2：访客端 MVP（微信 H5 展示优先）
实现页面与路由：
- `GET /` 首页（轮播 + 优势 + 产品列表 + 案例 + 门店信息 + CTA）
- `GET /categories/:categoryId` 分类页
- `GET /products/:productId` 产品详情页
- `GET /contact` 联系页（电话 tel 链接 + 微信咨询入口 + 地址/导航）
- `GET /inquiry` 线索表单页（先放 UI，接口先按后续补齐）

CTA 行为（按优先级）：
- 微信咨询：二维码扫码 + 微信号复制添加
- 拨打电话：兜底入口（`tel:`）

可验证测试点
- 在手机浏览器/微信内置浏览器打开能正常渲染
- CTA 可点击，跳转正确（tel/微信号/二维码区域）

### Step 3：管理端 MVP（内容能维护）
实现最小可用管理功能（单用户）：
- 登录/权限
- 轮播图管理（上下架/启用）
- 分类增改删
- 产品增改删（图片先可用占位上传或简单上传）
- 门店基础配置（电话/地址/微信咨询信息/SEO）
- （可选）案例/评价管理

可验证测试点
- 管理端新增产品后，访客端对应详情可见
- 轮播/分类开关生效

### Step 4：图片验证码（后端生成）+ 线索闭环
按你要求的防刷规则实现：
1. 获取验证码
   - `GET /api/captcha`
   - 返回：`captchaId` + `captchaImage`（base64 或可直接渲染的图片 URL）
2. 提交线索（MVP 表单字段）
   - 表单字段：`phone` 必填；`name/region/description` 可选
   - 防刷字段：`captchaId` + `captchaCode`
   - 接口：`POST /api/inquiries`（body：`phone + captchaId + captchaCode` + 可选字段）
3. 过期策略
   - `captchaId` 过期返回文案：`验证码过期请重新获取并刷新图片`

管理端线索管理 MVP：
- `GET /api/admin/inquiries` 列表（按状态筛选可选）
- `PUT /api/admin/inquiries/:id` 修改状态（NEW/CONTACTED/IGNORED）

可验证测试点
- 获取验证码->填写正确验证码->提交成功
- 使用过期 captchaId 提交：前端提示“验证码过期请重新获取并刷新图片”
- 提交成功后管理端能看到线索记录（字段与时间一致）

### Step 5：微信 H5 分享与体验收尾

性能/兼容：

- 移动端按钮区域足够大，弱网下避免长时间白屏

可验证测试点

- 微信内打开页面没有明显跳转/布局错乱

# 项目待办事项 (todo.md)

根据 `dev_order.md` 与 `plan.md` 的规划，以下是项目中尚未完成或需要完善的功能点。

## 1. 基础架构与环境 (Step 0 & Step 1)
- [ ] **数据库持久化**：当前后端使用 `InMemoryCatalogService` 和 `InquiryService`（内存存储），重启后数据丢失。需要接入 MySQL 数据库。
- [ ] **环境配置统一**：完善 `application.yml` 或 `.env`，确保数据库连接、文件上传地址、站点域名等配置可灵活切换。

## 2. 访客端功能完善 (Step 2 & Step 5)
- [ ] **图片资源真实化**：当前前端大量使用 SVG 占位图（如产品详情页、案例页）。需要实现真实的图片展示逻辑。
- [ ] **地图导航精准化**：`ContactPage.vue` 中的地图链接目前跳转到 Google Maps 搜索，国内环境建议适配高德/百度地图。
- [ ] **SEO 与微信分享优化**：
    - [ ] 确保每个页面都有完善的 `title/description/og:image`。
    - [ ] 在真机微信环境下验证分享卡片效果。
- [ ] **性能优化**：实现图片懒加载、资源压缩，确保弱网环境下的加载速度。

## 3. 管理端功能完善 (Step 3 & Step 4)
- [ ] **图片上传功能**：
    - [ ] 后端实现 `POST /api/media/upload` 接口，支持图片上传到本地服务器或对象存储（OSS）。
    - [ ] 前端管理端集成上传组件，替换目前的手动输入 `imageId`。
- [ ] **案例/评价管理**：
    - [ ] 后端补全 `CaseReview` 的增删改查管理接口。
    - [ ] 前端管理端增加“案例管理”标签页。
- [ ] **内容管理增强**：
    - [ ] 产品管理：支持多图上传与排序。
    - [ ] 门店配置：支持上传微信二维码图片。
- [ ] **权限与安全性**：
    - [ ] 目前使用硬编码的 `dev-admin-token`。需要实现真正的登录态校验（如 JWT）和密码加密存储。

## 4. 联调与部署 (Step 6)
- [ ] **全链路闭环测试**：从访客提交线索到管理端处理状态的完整流程测试。
- [ ] **生产环境部署**：
    - [ ] 配置 HTTPS（微信 H5 必须）。
    - [ ] 域名备案与微信 JS 域名配置（如需高级分享功能）。
- [ ] **数据初始化**：准备 50-200 个产品的真实数据进行压测和展示验证。

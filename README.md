## 农村地区红木家具广告网站（微信端展示）

本项目面向农村地区用户，提供一个“红木家具品牌/门店宣传 + 线上获客”的广告型网站，并确保能在微信端稳定展示（包括微信内浏览器/H5 场景、以及面向微信分享的页面体验）。

### 为什么要适配农村地区 + 微信使用习惯

1. 农村用户对网站复杂交互接受度较低，因此需要“少步骤获取信息、按钮直达咨询/致电/定位”。
2. 微信是高频入口，用户往往在微信内打开页面并进行分享、转发、咨询。
3. 网络环境可能较慢，因此需要图片懒加载、体积控制、关键内容优先渲染，避免长时间白屏。

### 核心目标

- 展示红木家具品类、工艺与门店优势（可通过图片/短文/案例增强信任）。
- 让用户在微信端快速完成咨询：点击拨打电话、发送微信咨询、提交需求表单。
- 让门店能后台维护内容：产品上架、分类、轮播图、活动、客户咨询等。

### 主要用户角色

- 浏览者（农村用户/潜在客户）：查看产品/案例/地址信息，发起咨询。
- 咨询者（已填写或已联系的用户）：查看回复/预约状态（可选）。
- 门店管理员（运营人员/老板）：维护网站内容与接收线索。

### 关键使用场景（微信端）

1. 微信聊天/群聊中收到分享链接，打开即能看到清晰的品牌介绍与“立即联系”入口。
2. 用户看到感兴趣的产品后，能一键复制/拨打电话、或跳转到微信联系（根据实际形态：公众号/小程序/外链）。
3. 用户需要了解地理位置时，页面提供“地图导航入口”（打开微信内地图或系统地图）。

### 功能需求（MVP 建议）

#### 访客端（微信可见）

- 首页
  - 顶部轮播/主视觉海报
  - 品牌介绍（工艺/材料/售后承诺）
  - 特色产品推荐（列表卡片）
  - 客户评价/案例展示
  - 门店信息（电话、地址、导航、营业范围）
  - CTA：`微信咨询（添加微信：二维码扫码 + 微信号复制添加）`、`拨打电话`、`提交需求`
- 产品/分类页
  - 分类筛选（如：餐桌椅、柜类、床/床头、茶台、屏风等——按实际填）
  - 产品详情（图片画廊、尺寸/材质/工艺说明、适用场景）
  - 产品详情页 CTA 同首页
- 线索提交
  - 表单字段：姓名（可选）、电话、所在地区（可选）、需求描述
  - 提交后展示“提交成功/等待联系”的状态页
- 关于我们
  - 品牌故事、工厂/门店介绍、服务流程（选填）
- 联系我们
  - 电话一键拨号、微信联系入口、地址与地图链接

#### 管理端（门店运营）

- 内容管理
  - 轮播图、首页模块开关、产品分类、产品信息（图文）
  - 客户评价/案例（可审核或直接展示，取决于策略）
- 线索管理
  - 查看用户提交的咨询记录、状态（新建/处理中/已联系）
  - 基础导出（CSV 导出可选）
- 基础配置
  - 电话、地址、微信联系方式（微信号 + 微信二维码）
  - SEO/分享文案（标题、描述、封面图）

### 微信展示策略（需要在实现前确认）

本项目已确定使用 **微信 H5**（在微信内直接打开网页）：

- 优点：开发成本相对低，直接复用 Web 技术栈
- 重点：移动端适配、弱网加载体验、微信分享卡片（通过 `OG meta` 生成默认预览）

### 非功能需求（建议）

- 性能：微信内页面首屏尽量在弱网条件下稳定可用（建议控制图片体积与懒加载）。
- 兼容：移动端宽屏自适应、按钮区域足够大（方便点击）。
- 安全：表单防刷（图片验证码验证：有效期 1 分钟，可配合简单限流可选）。
- 可维护：内容由管理员在后台维护，无需频繁改代码。
